# Руководство по тестированию платформы

## Тестирование различных ролей пользователей

### Вариант 1: LayoutSwitcher (Быстрое тестирование)

**Преимущества:**
- Мгновенное переключение между ролями
- Нет необходимости в регистрации новых пользователей
- Удобно для быстрой проверки интерфейса

**Недостатки:**
- Не тестирует реальные RLS политики
- Не проверяет взаимодействие между пользователями
- Только эмуляция интерфейса

**Использование:**
1. Войдите как разработчик (Oleg Shkola)
2. В сайдбаре найдите блок "Тестовый режим"
3. Выберите нужную роль: Клиент, Исполнитель, Админ
4. Интерфейс изменится согласно выбранной роли

**Недостатки:**

- Необходимо перелогиниваться для смены роли

## Чек-лист тестирования

### Для каждой роли проверить:

#### Общие функции:
- [ ] Авторизация
- [ ] Просмотр профиля
- [ ] Редактирование профиля
- [ ] Навигация по секциям

#### Клиент (`client`):
- [ ] Доступ к клиентским секциям
- [ ] Создание чатов
- [ ] Отправка сообщений
- [ ] Просмотр сделок
- [ ] Заказ услуг

#### Исполнитель (`performer`):
- [ ] Доступ к секциям исполнителя
- [ ] Создание предложений
- [ ] Управление заказами
- [ ] Статистика работы

#### Администратор (`ww_admin`):
- [ ] Доступ ко всем секциям
- [ ] Модерация чатов
- [ ] Управление пользователями
- [ ] Редактирование/удаление чатов
- [ ] Системная аналитика

## Сценарии тестирования

### Быстрое тестирование интерфейса (LayoutSwitcher):
1. Войдите как разработчик
2. Переключайтесь между ролями
3. Проверьте видимость нужных секций
4. Убедитесь в корректности отображения элементов

### Полное тестирование (отдельные аккаунты):
1. Зарегистрируйте тестовых пользователей
2. Создайте тестовые чаты между пользователями
3. Проверьте права доступа к данным
4. Протестируйте взаимодействие между ролями

## Примечания для разработчиков

- Используйте LayoutSwitcher для быстрой проверки UI
- Создавайте отдельных пользователей для тестирования бизнес-логики
- Проверяйте консоль браузера на ошибки авторизации
- Тестируйте RLS политики с реальными пользователями
